# Vercel Deployment Guide for Alerto360

## Quick Deploy Steps:

### 1. Deploy via Vercel Dashboard
1. Go to https://vercel.com/dashboard
2. Click "Add New..." → "Project"
3. Import from GitHub: `RoelMorato/Alerto360_Main`
4. Framework: "Other" (PHP project)
5. Click "Deploy"

### 2. Add Environment Variables
After deployment, go to Project Settings → Environment Variables:

```
GROK_API_KEY=your-xai-api-key-here
GEMINI_API_KEY=your-gemini-api-key-here
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-gmail-app-password
```

### 3. Database Setup
Vercel doesn't support MySQL directly. Options:

**Option A: Vercel Postgres (Recommended)**
- Go to Storage tab in Vercel dashboard
- Create Postgres database
- Update `api/db_login.php` with Vercel Postgres credentials

**Option B: External Database**
- Use PlanetScale, Railway, or Supabase
- Update connection details in `db_connect.php`

### 4. Update Flutter App
After deployment, update your Flutter app:

1. Open `alerto360_app/lib/services/api_service.dart`
2. Change baseUrl to your Vercel URL:
```dart
static const String baseUrl = 'https://your-app-name.vercel.app';
```
3. Rebuild APK: `flutter build apk --release`

### 5. Test Deployment
- Visit: `https://your-app-name.vercel.app`
- Test login with: admin@alerto360.com / admin123
- Test mobile app with new URL

## Troubleshooting:

### PHP Runtime Issues
Vercel supports PHP but with limitations. If you encounter issues:
1. Use Vercel's PHP runtime
2. Ensure all PHP files are in root directory
3. Check Vercel function logs

### Database Connection
- Vercel functions are serverless - use connection pooling
- Consider using Vercel Postgres or external database
- Update connection strings in environment variables

### API Endpoints
- All API files should work as Vercel functions
- Test each endpoint: `/api_login.php`, `/api_register.php`, etc.
